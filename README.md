# CS370 Pirate Intelligent Agent
## Briefly explain the work that you did on this project. What code were you given? What code did you create yourself?
This project gave me a better understanding of how reinforcement learning actually works instead of just reading about it. I was provided with the basic project framework, including the maze, helper functions, and the code needed to run the environment. The structure was already in place, but the intelligent behavior of the pirate agent was not.
My responsibility was to complete the learning portion of the project. I implemented the Deep Q Network (DQN) logic, added the epsilon greedy strategy for exploration, trained the neural network using experience replay, and updated the target network as the training progressed. I also spent time testing different settings to see how they affected the agent's ability to consistently reach the treasure. Watching the agent improve over time helped me understand how reinforcement learning develops through repeated experience instead of following fixed instructions.
## What do computer scientists do and why does it matter?
Computer scientists solve problems by designing software that helps people complete tasks more efficiently and securely. Their work is used in almost every industry, from healthcare and banking to transportation and artificial intelligence. Behind nearly every application that people use each day is someone who designed, tested, and improved the software until it was reliable enough to trust.
## How do I approach a problem as a computer scientist?
I like to understand the problem before I start writing code. Once I know what the program is supposed to accomplish, I break it into smaller pieces that are easier to test. During this project I learned that making small changes, testing the results, and adjusting from there was much more effective than trying to solve everything at once. Reinforcement learning especially showed me that improvement comes from analyzing results and making informed changes over time.
## What are my ethical responsibilities to the end user and the organization?
I believe developers have a responsibility to build software that people can trust. That means writing code that is secure, dependable, and designed with the user in mind. Organizations also depend on developers to create systems that are maintainable and accurate because mistakes can affect both the company and its customers. As artificial intelligence continues to grow, it is important to think about fairness, privacy, and making sure automated systems are used responsibly instead of relying only on performance.
